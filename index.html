多拨和链路聚合

多拨和链路聚合

一、多播（Multi-WAN）

核心目标：通过建立多个独立的WAN连接（如PPPoE拨号），突破单条宽带线路的带宽限制，提升外网带宽。分为虚拟多拨和实线多拨。

1. 虚拟多拨，这也是目前使用最多的一种方式，但已经有了较大限制。

  • 原理：拿少数的宽带和账号，使用 OpenWRT等固件模拟多个端口在同一物理网口上并发拨号，从而获得更多的“虚拟”宽带，随后进行负载均衡，以此获得网速叠加。需要虚拟化技术，是一种单线多拨。
  • **限制：**多数运营商会检测并发会话，采取以下限制：
    • 带宽共享：多拨后总带宽仍为单线上限（如500Mbps双拨仍为500Mbps）。
    • 会话数限制：强制后拨连接踢掉先拨连接，导致仅单线生效。
    • 账号封禁：频繁多拨可能触发风控，临时冻结账号。

2. 实线多拨，是一种稳定的合规的多拨形式。

  • **原理：**需要多个宽带账号，使用多根物理网线，这些网线可以来自同一运营商，也可以来自不同运营商。然后通过负载均衡设备，例如软路由，把多条线路的带宽进行整合。这种方式通常不需要依赖虚拟技术，而是直接通过多个物理接口来实现带宽叠加。

1. 应用场景

  • 家庭/小型企业：运营商宽带套餐有带宽上限（如500Mbps），通过多拨叠加到1Gbps。
  • 多ISP冗余：同时接入电信和联通宽带，避免单运营商故障导致断网。

2. 实际效果

  • 带宽叠加条件：
    • 运营商策略开放：允许同一账号并发拨号且不限制带宽池（如某些地区的电信宽带）。
    • 测速验证：通过多线程下载或SpeedTest测试，若双拨后速度接近翻倍则为成功。
  • 典型案例：
    • 用户使用OpenWRT路由器，单线双拨后，下载速度从500Mbps提升至900Mbps（接近物理网口上限1Gbps）。

3. 局限性

  • 运营商封堵：部分运营商会检测并发拨号，强制下线多余连接（如移动宽带常见）。

  • IP地址冲突：不同WAN的公网IP可能影响部分服务（如在线游戏、远程访问）。

二、链路聚合（Link Aggregation）

核心目标：将多个物理端口绑定为单一逻辑通道，提升内网设备间传输带宽和可靠性。

  • 原理：
    • 流量分发：基于MAC地址、IP或端口号的哈希算法分配流量。
    • 冗余容错：某条物理链路故障时，流量自动迁移至其他链路。

1. 应用场景

  • 服务器与交换机互联：通过4x1Gbps链路聚合实现4Gbps带宽，避免NAS或服务器成为瓶颈。
  • 核心网络设备互联：交换机间通过聚合链路提升主干带宽。

2. 实际效果

  • 带宽提升：
    • 聚合后的总带宽=单端口带宽×可用链路数（如2x1Gbps聚合后理论带宽2Gbps）。
    • 注意：单条TCP连接仍受限于单端口带宽（如单个文件下载最高1Gbps）。
  • 典型案例：
    • 企业使用支持LACP的交换机和NAS，通过2条10Gbps光纤聚合，实现20Gbps内网传输。

3. 局限性

  • 硬件要求：需交换机、路由器、终端设备均支持聚合协议。

  • 配置复杂度：需两端设备匹配聚合模式（LACP/静态）。

三、两者对比

特性 多拨（Multi-WAN） 链路聚合（Link Aggregation）
应用场景 外网（WAN侧），通过多个拨号连接提升带宽 内网（LAN侧），聚合交换机或设备间物理链路
技术原理 基于PPPoE多次拨号，获取多个公网IP地址 基于LACP（动态聚合）或静态绑定，合并多个端口
带宽叠加条件 依赖运营商策略（是否允许并发拨号及带宽叠加） 依赖两端设备均支持聚合协议（如LACP）
IP地址 每个拨号连接独立公网IP 聚合后的逻辑接口共享同一IP或MAC地址
典型用途 家庭/小型企业突破单线带宽限制 数据中心/企业内网提升服务器或交换机间带宽

穿透和组网的网络拓扑思路

穿透和组网的网络拓扑思路

代理模式内网穿透

使用NPS（或FRP），利用反向代理，实现内网穿透（即把一台内网中设备暴露到公网中去）

设备要求：

  1. 用于桥接的公网服务器。（可以为云服务器）

  2. 一台内网中提供服务的服务器。

配置流程（以云服务器为例）

  1. 使用workbench浏览器或第三方SSH工具（如WinSCP、PuTTY、XShell）连接到云服务器。
  2. 在桥接服务器（任意带公网ip的服务器，例如云服务器）下载nps（或frp,github中即可下载）服务端，并按要求配置服务器信息，启动服务端程序。（输入命令 ./nps start）
  3. 访问桥接服务器IP地址:端口号进入管理界面，尝试登录nps，检测是否成功部署服务端程序。
  4. 进入nps管理界面（即**3.**中的网址），添加一个客服端，复制这个客户端对应的带有密钥的命令。
  5. 在需要被穿透、暴露在公网的服务器中下载nps客户端。在nps文件夹的可执行文件(nps.exe)文件夹中（或上一级文件夹），启动gitbash命令行，执行**4.**中复制的命令。
  6. 检查nps服务界面，服务器是否成功上线。失败检查UPnP（默认开启），防火墙等。
  7. 在tcp(udp)隧道中，添加转发规则。服务端端口为任意空端口（0~65535）,目标（即本地ip地址与端口）填写本地路由转发后的ip地址（ipconfig即可查看本地Ip地址，端口填写对应开放软件的端口），客户端ID为**4.**中添加的客户端ID。

代理模式P2P隧道配置

使用nps，在经过反向代理后，直接打通隧道，后续流量不需要桥接服务器转发，可以直接传输。

设备要求：与内网穿透相同。

配置流程：

  1. 基础配置参见内网穿透反向代理部分。
  2. 重新配置服务端的配置文件。（添加p2p_ip和p2p_port）
  3. 在nps管理界面中，添加一个p2p代理，目标（IP:端口）填写局域网中要想连接的设备IP地址，端口填写对应软件需要端口。客户端ID与创建客户端时保持一致，密钥填写任意字符。复制带有密钥的命令。
  4. 将**3.**中复制的密钥在客户端执行。
  5. 参照内网穿透部分6. **7.**部分，验证是否部署成功。

家庭VPN配置 （使用L2TP协议）

设备要求：

  1. 具有公网IP的服务器。（如果没有可以参照代理模式的内网穿透进行配置）
  2. 一台具有VPN功能的路由器（或选择软路由），并且路由器处于路由模式，而不是光猫下的桥接（公网ip必备条件）。

配置流程：（以家用动态公网IP，路由设置为爱快软路由为例）

  1. 购买/租用域名，使用DDNS（动态域名解析）将动态IP地址变为稳定的域名。
  2. 开启L2TP服务，设置密钥。
  3. 添加套餐和登录账号和密码。（不同路由管理器叫法不同）
  4. 在外网尝试是否能通过VPN（L2TP）登录，访问内网设备。

投资想法

投资笔记

投资应将长线和短线彻底划分开来，方法理念完全不相同，短线更应该叫做投机。

长线投资

进行长线投资主要以高股息股票为主。年化利率为股息支付率(%)÷PE,年化收益率通常超过6%。

长线投资时可以分为两大类，周期性类与大盘类。

  1. 周期性类

  • 常见行业：大宗商品，如煤炭，钢铁，船舶。

  • 行业特征：产品价格呈周期性波动的，产品的市场价格是企业赢利的基础。 在市场经济情况下，产品价格形成的基础是供求关系，而不是成本，成本只是产品最低价的稳定器，但不是决定的基础。

  • 投资思路：尝试分析未来供给需求，买入后长期持有，可以不断少量补仓和持续定投，直至不看好行业后卖出。

  1. 跟随大盘类
  • 常见行业：日用品。
  • 行业特征：产品供需关系稳定，企业经营良好，但发展前景不明朗，想象空间差，股价较为稳定，跟大盘关系紧密。
  • 投资思路：在大盘表现差的时候买入（大盘表现差为，指数与现实发展基本面结合，指数低于基本面预期），在真正触底前不要轻易加仓，当大盘高于基本面预期时，逐步减仓，减仓后不要再轻易买入。

短线投资

短线投资追求快进快出，及时止损，超短线应该按照计划走，不上头，不留恋。埋伏型短线时间也通常不超过一年。

短线投资可以分为追热点型短线和埋伏型短线

  1. 追热点短线
  • 追逐当前热点，判断短期形式，风险高，亏钱概率高，不适合大多数人。可以通过淘股吧等软件适当参考大佬方向，适当抄作业。
  1. 埋伏短线
  • 短线投资应该以埋伏式为主，只看好自己熟知的行业，建立对他人信息差来埋伏相关产业，在大盘危险的时候，不适合立马投入，注意规避风险。

内网穿透和异地组网

内网穿透原理

在实现内网穿透的方法中，对NAT设备的控制需求和是否需要第三方桥接情况各有不同，以下是具体分析：

需要控制NAT设备的方法

  • 端口映射：此方法需要在NAT设备（如路由器）上进行手动配置，明确地将外部端口与内部设备的端口进行对应设置，以允许外部请求能正确到达内网设备，所以必须要对NAT设备有控制权限才能进行相关配置操作。
    • 原理：在路由器或防火墙等网络设备上进行设置，将外部网络的特定端口与内部网络中特定设备的特定端口进行映射。当外部网络的请求到达路由器的公网 IP 的指定端口时，路由器会根据设置将请求转发到内部网络中对应设备的相应端口上，从而实现外部对内部设备的访问。
    • 示例：假设内网中有一台服务器，其 IP 地址为 192.168.1.100，提供 Web 服务，端口为 80。通过在路由器上设置端口映射，将路由器的公网 IP 的 8080 端口映射到内网服务器的 80 端口。这样，外部用户访问路由器的公网 IP 的 8080 端口时，实际上访问的是内网中 IP 为 192.168.1.100 的服务器的 80 端口。
  • NAT穿越技术：在一些情况下，尤其是当需要实现特定的、更灵活的内网穿透功能时，可能需要对NAT设备进行配置调整，比如设置特定的NAT转换规则、调整端口范围等，以确保内网设备能顺利与外网进行通信，因此也需要一定程度上控制NAT设备。
    • 原理：NAT 设备会为内部网络中的设备分配私有 IP 地址，并维护一个地址转换表，记录内部私有 IP 地址与外部公网 IP 地址之间的映射关系。当内部设备要与外部设备通信时，NAT 设备会将内部设备的私有 IP 地址转换为公网 IP 地址，使数据包能够在公网上传输。同时，当外部设备响应时，NAT 设备又会根据地址转换表，将公网 IP 地址转换回内部设备的私有 IP 地址，将数据包正确地转发给内部设备。
    • 示例：家庭网络中的多台设备通过路由器连接到互联网，路由器就是一个 NAT 设备。当家里的电脑要访问互联网上的网站时，电脑使用的是内部私有 IP 地址，路由器会将电脑发出的数据包中的源 IP 地址转换为路由器的公网 IP 地址，然后发送到互联网上。当网站响应时，路由器再根据之前的映射关系，将数据包中的目的 IP 地址转换为电脑的私有 IP 地址，将数据正确地交给电脑。

需要第三方桥接的方法

  • 反向代理：通常需要在公网中部署一台第三方的反向代理服务器，该服务器作为外部请求与内网服务之间的桥梁。外部用户的请求先到达反向代理服务器，再由它转发到内网服务器，然后将响应返回给外部用户，这里的反向代理服务器就是起到了第三方桥接的作用。
    • 原理：在公网中有一台具有公网 IP 的服务器作为反向代理服务器。内网中的服务器将自己的服务信息注册到反向代理服务器上。当外部用户请求访问内网中的服务时，先将请求发送到反向代理服务器，反向代理服务器根据请求的内容和事先的配置，将请求转发到对应的内网服务器上，然后再将内网服务器的响应返回给外部用户。
    • 示例：企业内部有多个业务系统，如财务系统、办公系统等，都在内网中。在公网上部署一台反向代理服务器，企业的域名解析到这台反向代理服务器。当员工在外部访问企业的财务系统时，请求先到达反向代理服务器，反向代理服务器根据请求的路径等信息，将请求转发到内网中的财务系统服务器上，获取数据后再返回给员工。
  • VPN（虚拟专用网络）：一般需要借助VPN服务器来建立连接，VPN服务器作为第三方，在用户和内网之间搭建起虚拟通道。用户通过VPN客户端与VPN服务器连接，经身份验证后获得虚拟IP地址来访问内网资源，VPN服务器起到了桥接内外网络的作用。
    • 原理：通过在公网上建立一条虚拟的专用通道，将内网和外网连接起来。用户通过 VPN 客户端与 VPN 服务器建立连接，VPN 服务器验证用户身份后，为用户分配一个虚拟的 IP 地址，使用户能够通过这个虚拟 IP 地址访问内网资源，就好像用户直接连接到了内网一样。
    • 示例：企业员工出差在外，需要访问企业内部的办公系统。员工在自己的笔记本电脑上安装 VPN 客户端软件，通过 VPN 客户端连接到企业的 VPN 服务器，经过身份验证后，员工的电脑就可以获得一个企业内网的虚拟 IP 地址，从而可以像在企业内部一样访问办公系统等各种内网资源。
  • 隧道协议：通常需要有一个位于公网的第三方设备或服务器来协助建立隧道。发送端将数据封装在隧道协议数据包中发送给第三方设备，第三方设备再根据隧道协议的规则将数据转发到目标内网，在接收端再进行解封装，这里的第三方设备起到了桥接不同网络的作用，使数据能够通过隧道在不同网络之间传输。
    • 原理：将一种网络协议的数据封装在另一种网络协议的数据包中进行传输，在不同网络之间建立起一条逻辑上的隧道。通过这种方式，使得原本不能在公网上直接传输的内网协议数据能够在公网上传输，就好像在公网上开辟了一条专门为内网数据传输的通道。
    • 示例：在企业网络中，可能使用了某种特定的企业内部网络协议来进行数据通信。为了让位于不同地点的企业分支机构之间能够相互通信，就可以使用隧道协议，如 IPsec 隧道模式。将企业内部协议的数据封装在 IPsec 数据包中，通过公网进行传输。在接收端，再将数据从 IPsec 数据包中解封装出来，还原为企业内部协议的数据，供相应的设备和应用程序使用。

异地组网与内网穿透(可能工具nps)

内网穿透和异地组网既有一定的关联，也存在明显的区别，以下是具体介绍：

关系

  • 目标有重叠：内网穿透和异地组网的最终目的在某些方面是一致的，都是为了实现不同网络环境下设备之间的通信和资源共享。例如，企业可能需要通过内网穿透让外部人员访问内部服务器资源，也可能需要通过异地组网将不同分支机构的网络连接起来，实现资源共享和协同工作。
  • 技术有交集：部分技术手段在两者中都会用到，如VPN技术、端口映射等。在实现内网穿透时，可以利用VPN建立虚拟通道，使外部设备能访问内网资源；在异地组网中，VPN也是常用的技术之一，用于在不同地域的网络之间建立安全的连接。
  • 相互辅助：内网穿透可以作为异地组网的一种补充手段。在已经建立的异地组网环境中，如果某些特殊设备或服务需要更灵活的外部访问方式，就可以通过内网穿透来实现。例如，在企业异地组网的基础上，可能还需要让合作伙伴能够访问企业内部的特定测试服务器，这时就可以利用内网穿透技术来满足需求。

区别

  • 应用场景不同
    • 内网穿透：主要用于将位于内网中的单个或多个特定设备、服务暴露给外网，使外网用户能够访问到这些内网资源。常见场景如个人用户在家中搭建了一个私人网站服务器，希望通过互联网让其他人能够访问，就可以使用内网穿透技术。
    • 异地组网：侧重于将不同地理位置的多个网络连接成一个逻辑上的统一网络，使不同网络中的设备能够像在同一个局域网中一样进行通信和资源共享。典型场景是企业的多个分支机构分布在不同城市，需要将这些分支机构的网络连接起来，实现数据共享、视频会议等功能。
  • 网络结构不同
    • 内网穿透：重点在于突破单个网络的边界，实现外网对内网特定资源的访问，通常是从外网的一个点到内网的一个或多个点的连接。其网络结构相对简单，主要是在现有网络基础上，通过一些技术手段打通内外网之间的通道。
    • 异地组网：涉及多个不同地点的网络之间的连接和整合，形成一个更大规模的网络结构。这些网络可能有各自的子网、路由器等设备，需要通过特定的技术将它们连接起来，构建一个复杂的网络拓扑，可能包括多个子网之间的路由配置、网络地址转换等操作。
  • 技术实现侧重点不同
    • 内网穿透：更注重如何突破NAT限制，将内网中的端口或服务映射到外网，使外部能够访问到内部资源。常用技术有端口映射、反向代理等，主要解决的是内外网之间的单点通信问题。
    • 异地组网：更关注不同网络之间的整体连接和通信，需要考虑网络的稳定性、安全性、可扩展性等多个方面。除了使用VPN等技术建立连接外，还需要进行网络规划、路由配置、安全策略设置等一系列工作，以确保整个网络的正常运行和数据传输。

路由器小结

路由器DMZ

路由器的DMZ（Demilitarized Zone，非军事区）是一种网络安全区域设置，它是为了解决安装防火墙后外部网络无法访问内部网络服务器的问题而设立的一个非安全系统与安全系统之间的缓冲区。以下是关于它的详细介绍：

作用

  • 提供特定服务访问：允许外部网络有限度地访问内部网络中的特定服务器或设备。比如企业希望对外提供Web服务、FTP服务等，又不想让整个内部网络暴露在外部风险下，就可以把提供这些服务的服务器放置在DMZ区域。这样，外部用户可以访问DMZ中的服务器获取相应服务，但无法直接访问内部网络的其他资源，在一定程度上保障了内部网络的安全。

  • 隔离与保护内部网络：将对外服务的设备与内部核心网络隔离开来。如果DMZ中的服务器受到攻击，攻击者也较难进一步渗透到内部网络，从而为内部网络中的数据和设备提供了额外的安全防护层。

工作原理

  • 端口映射：路由器通过配置端口映射规则，将外部网络请求的特定端口映射到DMZ区域内设备的相应端口。例如，当外部用户访问服务器的80端口（通常用于HTTP服务）时，路由器会将这个请求转发到DMZ中Web服务器的80端口，使外部用户能够访问到Web服务器上的网页内容。
  • 访问控制：路由器或防火墙会根据预设的访问控制列表（ACL）来决定哪些外部流量可以进入DMZ，以及DMZ中的设备可以访问哪些外部资源和内部网络资源。一般来说，会允许外部对DMZ中特定服务端口的访问，同时限制其他不必要的访问，以确保只有合法的流量能够通过。

设置步骤

不同品牌和型号的路由器设置DMZ的步骤可能会有所不同，但一般来说可以按照以下基本步骤进行：

  1. 登录路由器管理界面：在浏览器中输入路由器的管理地址，通常是192.168.1.1或192.168.0.1等，然后输入用户名和密码登录。
  2. 找到DMZ设置选项：在路由器管理界面中，查找“DMZ”“虚拟服务器”或“转发规则”等相关选项。
  3. 配置DMZ主机：在DMZ设置页面，输入要设置为DMZ的设备的IP地址。这个IP地址通常是内部网络中需要对外提供服务的服务器或设备的本地IP地址。
  4. 保存设置：完成IP地址输入后，点击“保存”或“应用”按钮，使设置生效。 虽然DMZ提供了一种让外部网络访问内部特定资源的便利方式，但也会带来一定的安全风险。因为DMZ中的设备相对更容易受到外部攻击，所以在使用DMZ时，需要谨慎配置访问规则，并确保DMZ中的设备本身具备足够的安全防护措施，如安装防火墙、杀毒软件等。
